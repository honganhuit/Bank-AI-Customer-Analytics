# BUSINESS REQUIREMENT DOCUMENT (BRD)

## 1. Project Overview

ATBank AI Customer Analytics is a system designed to support banking analysts in analyzing customer data, identifying churn risks, and segmenting customers to improve business decision-making.

---

## 2. Business Objectives

- Reduce customer churn by identifying high-risk customers
- Improve decision-making through data-driven insights
- Enhance customer segmentation for targeted marketing
- Increase efficiency in data analysis

---

## 3. Scope

### In Scope

- Upload and process customer datasets
- Validate data quality
- Provide data analysis and insights
- Perform customer segmentation
- Predict customer churn risk
- Display results in dashboard
- Export analysis results
- Manage user accounts

### Out of Scope

- Real-time data processing
- Integration with external systems
- Model customization
- Role-based access control

---

## 4. Stakeholders

- Bank Analyst
- Marketing Team
- Business Manager
- System Administrator

---

## 5. User Roles

| Role | Description                                    |
| ---- | ---------------------------------------------- |
| User | Analyze data, view dashboard, download results |

---

## 6. High-Level Workflow

Login → Upload → Validate → Analyze → Segment → Predict → View Results → Export

---

## 7. Key Features (Business View)

- User authentication
- Dataset upload and validation
- Data analysis and insights generation
- Customer segmentation
- Churn risk prediction
- Dashboard visualization
- Export results

---

## 8. Success Criteria

- Users can successfully upload and analyze datasets
- System provides meaningful insights
- Users can identify high-risk customers
- Results support business decisions

---

## 9. Assumptions

- Users have basic data knowledge
- Input data is structured
- System operates in offline mode

---

## 10. Constraints

- Limited to batch processing
- No integration with external systems
